# Minecraft 3D 版本设计与开发文档

## 1. 项目概述
本项目旨在基于现有的2D微型Minecraft项目，在 `d:\github\minecraft9\source3d` 目录下从零构建一个基于Python的3D版本Minecraft原型。
项目核心目标是实现基础的体素生成、方块交互、简单的物理规则以及生存模式的基础闭环。

**底层引擎**: Ursina Engine (基于 Panda3D)

## 2. 核心技术点
*   **体素生成 (Voxel Generation)**: 使用正弦/余弦波叠加等数学函数生成起伏的程序化地形。
*   **区块管理 (Chunk Management)**: 实现按需加载和卸载玩家周围 `16x16` 的区块空间，达到伪无限世界的效果。
*   **网格合并优化 (Mesh Combine)**: 为解决大量 Entity 带来的严重卡顿，将同类型方块的表面合并为单一网格（Mesh）。手动计算可见面，极大降低了渲染调用（Draw Calls）。
*   **背面剔除与顶点环绕顺序 (Backface Culling & Winding Order)**: 确保所有网格表面的顶点顺序为顺时针 (CW)，生成正确的法线朝向以解决视觉空洞和透视问题。
*   **射线检测 (Raycasting)**: 用于第一人称视角的方块挖掘、放置目标定位，以及实体碰撞和视线遮挡排除。
*   **UV 贴图映射与像素过滤 (UV Mapping & Point Filtering)**: 将2D原版贴图映射到3D网格表面，并关闭纹理过滤以保持Minecraft特有的清晰像素画风格。
*   **状态机 (State Machine)**: 用于实现动物AI，使其能在 `idle`（空闲）和 `moving`（游荡）状态间切换。

## 3. 核心功能模块与代码结构
*   **主控与物理 (`main.py`)**: 
    *   包含游戏的核心主循环、输入处理、第一人称控制器和UI。
    *   实现了自定义的 `PlayerController`（包含重力、水下浮力缓冲、摔落伤害）。
    *   处理物品拾取与背包同步更新。
*   **世界生成 (`world.py`)**: 
    *   负责地形算法和基于贴图的区块网格合并逻辑。
    *   包含树木、水体和动物的生成机制。
*   **实体系统 (`entity.py`)**: 
    *   包含生存交互核心逻辑。
    *   `Animal` 类：游荡AI、重力与浮水、碰撞检测、受击判定。
    *   `ItemDrop` 类：受重力影响的旋转掉落物（如肉块）。
*   **数据持久化 (`save_system.py`)**: 
    *   将玩家位置、方块改动、背包物品数量（`inventory`）和生命值（`health`）保存到本地 JSON 文件中。
*   **常量配置 (`constants.py`)**: 
    *   存储方块类型、热栏列表、世界尺寸、水位线等。

## 4. 问题修复与踩坑记录
在开发过程中，我们解决了一系列关键性问题：
1.  **白屏/画面全白 (Overexposure)**
    *   *原因*: `Ursina` 的 `color.rgba()` 接收 0.0~1.0 的浮点数，直接传入 0-255 导致极致过曝。
    *   *修复*: 将所有 RGB 值除以 255.0。
2.  **完全看不见地图 (掉入虚空)**
    *   *原因*: 早期测试时玩家掉入虚空，极深的 Y 坐标（如 -23857）被保存到了存档中。
    *   *修复*: 增加高度保护机制：若 `Y < -10` 则强制将玩家拉回地表并覆盖错误存档。
3.  **操作极度卡顿 (Entity Overhead)**
    *   *原因*: 每个方块创建一个独立的 Entity，成千上万的实体拖垮了 Python 和渲染管线。
    *   *修复*: 实现 `combine_chunk_mesh`，把同类型方块表面合并为单一网格。
4.  **方块透视/表面缺失**
    *   *原因*: 顶点连接顺序出错（CCW导致正面被剔除）。
    *   *修复*: 重新推导并统一所有 6 个面的顶点顺序为顺时针（CW）。
5.  **水下视野有奇怪掉落物阻挡视线**
    *   *原因*: 准星射线误碰到了水面动物或玩家自身模型。
    *   *修复*: 在射线检测的 `ignore_list` 中加入玩家、动物和高亮框实体。
6.  **打动物时游戏崩溃 (`AttributeError`)**
    *   *原因*: 重构拾取逻辑时，误删/遗漏了 `_check_item_pickup` 方法，导致 update 循环调用失败。
    *   *修复*: 在 `main.py` 中补回该方法并完善距离检测与背包计数逻辑。
